# Railway Setup Guide - Step by Step

**Objective:** Deploy FileHop (frontend + backend + database) on Railway

---

## Step 1: Create Railway Account & Project

1. Go to **railway.app**
2. Sign up / Log in
3. Click **"Create New Project"**
4. Choose **"Deploy from GitHub"**
5. Connect your GitHub account
6. Select repo: **SitlDev/FileHop**
7. Click **"Deploy Now"**

Railway will automatically:
- Create a PostgreSQL database
- Generate `DATABASE_URL` environment variable

---

## Step 2: Add Backend Service

**This adds the Express API server**

1. In Railway dashboard, click **"Add Service"** 
2. Select **"GitHub Repo"**
3. Choose repo: **SitlDev/FileHop**
4. **Service Settings:**
   - **Name**: `backend`
   - **Root Directory**: `backend`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`
5. Click **"Deploy"**

Wait for build to complete (2-3 minutes).

---

## Step 3: Add Frontend Service

**This adds the Next.js app**

1. Click **"Add Service"** again
2. Select **"GitHub Repo"**
3. Choose repo: **SitlDev/FileHop**
4. **Service Settings:**
   - **Name**: `frontend`
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Start Command**: `npm start`
5. Click **"Deploy"**

Wait for build to complete.

---

## Step 4: Configure Backend Environment Variables

1. Click on **backend** service
2. Go to **"Variables"** tab
3. Add these variables (copy-paste each):

```bash
# Server
NODE_ENV=production
PORT=5000

# Get FRONTEND_URL from Frontend service's public URL (see Step 5)
FRONTEND_URL=https://[WILL_UPDATE_AFTER_FRONTEND_DEPLOY]

# Database (auto-generated by Railway, just verify it's there)
DATABASE_URL=postgresql://...

# Auth - Generate new secrets:
JWT_SECRET=[GENERATE_NEW - use: openssl rand -base64 32]
NEXTAUTH_SECRET=[GENERATE_NEW - use: openssl rand -base64 32]

# AWS S3 / Backblaze B2 (from your account)
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=[YOUR_KEY]
AWS_SECRET_ACCESS_KEY=[YOUR_SECRET]
S3_BUCKET_NAME=filehop-upload
S3_ENDPOINT=s3.us-east-005.backblazeb2.com

# Stripe (LIVE KEYS - get from Stripe dashboard)
STRIPE_PUBLIC_KEY=pk_live_...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Email (Resend API - get from resend.com)
RESEND_API_KEY=re_...
RESEND_FROM_EMAIL=noreply@yourdomain.com

# Admin
ADMIN_EMAIL=admin@yourdomain.com

# File Settings
MAX_FILE_SIZE=524288000
USER_STORAGE_LIMIT=2147483648
FILE_RETENTION_DAYS=15
```

---

## Step 5: Configure Frontend Environment Variables

1. Click on **frontend** service
2. Go to **"Variables"** tab
3. Add these variables:

```bash
NEXT_PUBLIC_API_URL=https://[BACKEND_URL]
NEXT_PUBLIC_STRIPE_KEY=pk_live_...
```

**Where to get these URLs:**
- Go to backend service → click the URL in top right
- Copy the full URL (e.g., `https://filehop-backend-prod-xyz.railway.app`)
- Use that for `NEXT_PUBLIC_API_URL`

---

## Step 6: Get Service URLs

1. Click **backend** service
2. Look at top right - there's a **public URL** (e.g., `https://filehop-backend-xyz.railway.app`)
3. Click **frontend** service
4. Get its **public URL** (e.g., `https://filehop-frontend-xyz.railway.app`)

**Update backend variables:**
- Go back to backend → Variables
- Update `FRONTEND_URL` with the frontend URL
- Click **"Save"**

---

## Step 7: Run Database Migrations

The backend needs to set up the database schema:

1. Click **backend** service
2. Go to **"Logs"** tab
3. Wait for it to show "Server running on port 5000"
4. Go to **"Shell"** tab
5. Run:
   ```bash
   npm run db:migrate
   npm run db:seed  # Optional: adds sample data
   ```

Wait for both commands to complete.

---

## Step 8: Test Deployment

1. Open **frontend URL** in browser
2. You should see the FileHop landing page
3. Try to:
   - Sign up
   - Upload a file
   - See the download page with QR code
   - Test payment flow (use Stripe test cards if in test mode)

## Troubleshooting

### Backend won't start
- Check `npm run build` works locally
- Verify `DATABASE_URL` is set
- Check logs: **backend** → **Logs** tab

### Frontend can't reach backend
- Verify `NEXT_PUBLIC_API_URL` points to backend service URL
- Check CORS is enabled in backend
- Test API: `curl https://[backend-url]/api`

### Database connection fails
- Railway auto-creates `DATABASE_URL`
- Run migrations: `npm run db:migrate`
- Check Prisma schema is correct

### File uploads fail
- Verify S3/Backblaze B2 credentials are correct
- Check bucket name matches
- Test upload with small file first

## Deploy Flow

```
1. Create Railway Project
   ↓
2. Add Backend Service (builds & deploys)
   ↓
3. Add Frontend Service (builds & deploys)
   ↓
4. Set Environment Variables
   ↓
5. Run Database Migrations
   ↓
6. Test at https://[frontend-url]
```

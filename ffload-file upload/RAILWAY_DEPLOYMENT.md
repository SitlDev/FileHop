# FileHop - Single Railway Deployment Guide

**Objective:** Deploy frontend + backend + database on single Railway project

---

## Railway Project Architecture

```
FileHop (Single Railway Project)
├── Backend Service
│   ├── Source: GitHub (ffload-file upload/backend)
│   ├── Build: npm install && npm run build
│   ├── Start: npm start
│   ├── Port: 5000
│   └── Environment Variables: (see below)
│
├── Frontend Service
│   ├── Source: GitHub (ffload-file upload/frontend)
│   ├── Build: npm run build
│   ├── Start: npm start
│   ├── Port: 3000
│   └── Environment Variables: (see below)
│
└── PostgreSQL Database
    └── Auto-generated DATABASE_URL for backend
```

---

## Step 1: Create Railway Project

1. Go to **railway.app**
2. Click **"Create New Project"**
3. Choose **"Deploy from GitHub"**
4. Select repo: **SitlDev/FileHop**
5. Click **"Deploy Now"**

This creates the base project and PostgreSQL database automatically.

---

## Step 2: Add Backend Service

1. In Railway project, click **"Add Service"** → **"GitHub Repo"**
2. Select repo: **SitlDev/FileHop**
3. Settings:
   - **Root Directory**: `backend`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`
   - **Port**: `5000`
4. Click **"Deploy"**

---

## Step 3: Add Frontend Service

1. Click **"Add Service"** → **"GitHub Repo"**
2. Select repo: **SitlDev/FileHop**
3. Settings:
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Start Command**: `npm start`
   - **Port**: `3000`
4. Click **"Deploy"**

---

## Step 4: Configure Environment Variables

### Backend Service Variables

```bash
# Server
NODE_ENV=production
PORT=5000
FRONTEND_URL=https://[railway-frontend-url]

# Database (auto-generated by Railway)
DATABASE_URL=postgresql://user:pass@host/db

# Auth
JWT_SECRET=$(openssl rand -base64 32)
NEXTAUTH_SECRET=$(openssl rand -base64 32)

# AWS S3 / Backblaze B2
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=<your-key>
AWS_SECRET_ACCESS_KEY=<your-secret>
S3_BUCKET_NAME=<bucket-name>
S3_ENDPOINT=s3.us-east-005.backblazeb2.com

# Stripe (LIVE KEYS)
STRIPE_PUBLIC_KEY=pk_live_...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Email
RESEND_API_KEY=re_...
RESEND_FROM_EMAIL=noreply@yourdomain.com

# Admin
ADMIN_EMAIL=admin@yourdomain.com

# File Settings
MAX_FILE_SIZE=524288000
USER_STORAGE_LIMIT=2147483648
FILE_RETENTION_DAYS=15
```

### Frontend Service Variables

```bash
NEXT_PUBLIC_API_URL=https://[railway-backend-url]
NEXT_PUBLIC_STRIPE_KEY=pk_live_...  (same as backend)
```

## Step 5: Set Up Database Migrations

1. After backend deploys, run migrations:
   ```bash
   # In Railway backend service terminal:
   npm run db:migrate
   npm run db:seed (optional)
   ```

2. Or set up automatic migrations:
   - In backend service settings, add post-deploy hook:
   ```bash
   npm run db:migrate
   ```

---

## Step 6: Connect Services

Railway automatically creates internal networking:
- Backend can reach database via `DATABASE_URL`
- Frontend calls backend API via `NEXT_PUBLIC_API_URL` (external URL)
- Services communicate via Railway's internal network

---

## URLs After Deployment

- **Frontend**: `https://filehop-frontend-[id].railway.app`
- **Backend API**: `https://filehop-backend-[id].railway.app`
- **Database**: Internal only (not publicly exposed)

---

## Deployment Flow

1. Push code to GitHub
2. Railway detects changes
3. Triggers builds for all services
4. Deploys sequentially:
   - Database migrations (if configured)
   - Backend builds
   - Frontend builds
5. Services are live and connected

## Troubleshooting

**Backend won't start:**
- Check `npm run build` works locally
- Verify `DATABASE_URL` is set
- Check build logs in Railway

**Frontend can't reach backend:**
- Verify `NEXT_PUBLIC_API_URL` points to backend service
- Check CORS is enabled in backend
- Test API directly: `curl https://[backend-url]/api`

**Database connection fails:**
- Railway auto-creates `DATABASE_URL`
- Run migrations: `npm run db:migrate`
- Check Prisma schema is correct
